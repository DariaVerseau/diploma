# 🚀 Smart Image Lab — Веб-сервис для интеллектуальной обработки изображений

Веб-сервис для обработки изображений с использованием современных моделей машинного обучения: увеличение разрешения (Real-ESRGAN), перенос художественного стиля (AdaIN), восстановление портретов (CodeFormer) и классическая постобработка.

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/DariaVerseau/diploma.git
cd diploma
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
# Backend
SERVER_PORT=8080
DATABASE_URL=postgresql://user:password@db:5432/mydb?sslmode=disable
JWT_SECRET=your-secret-key-here-change-it-in-production
```

### 3. Запуск

```bash
# Собрать и запустить все сервисы
docker-compose up --build

# Или в фоновом режиме
docker-compose up -d
```

### 4. Проверка работы

| Сервис | URL | Статус |
|--------|-----|--------|
| Фронтенд | http://localhost:3000 | ✅ |
| API Gateway | http://localhost:8080/health | ✅ |
| ML-сервис | http://localhost:8000/health | ✅ |

### 5. Остановка

```bash
docker-compose down
```

## 📡 API Эндпоинты

### Аутентификация

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/v1/register` | Регистрация пользователя |
| POST | `/api/v1/login` | Вход в систему (JWT) |

### Изображения

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/v1/images` | Загрузка изображения |
| GET | `/api/v1/images` | Получение галереи (пагинация) |

### ML-операции

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/v1/ml/upscale` | Увеличение разрешения (2x/4x) |
| POST | `/api/v1/ml/style_transfer` | Перенос художественного стиля |
| POST | `/api/v1/ml/enhance` | Восстановление портретов |
| POST | `/api/v1/ml/postprocess` | Классическая постобработка |

### Управление аккаунтом

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/v1/change-password` | Смена пароля |

## ❓ Частые вопросы

### 1. ML-сервис долго запускается

ML-сервис загружает модели при первом запросе. Это может занять 30-60 секунд. Последующие запросы выполняются быстрее благодаря кэшированию.

### 2. Ошибка 413 Request Entity Too Large

Увеличьте `client_max_body_size` в `frontend/nginx.conf`:

```nginx
client_max_body_size 50M;
```

### 3. Недостаточно памяти на диске

```bash
# Очистка Docker
docker system prune -a --volumes -f
docker builder prune -a -f
```
